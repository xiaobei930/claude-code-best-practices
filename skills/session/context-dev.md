# 开发模式上下文

> 当 `/cc-best:mode dev` 激活时加载本文件，为开发模式注入行为指令。

## 行为指令

- 先写代码再解释，偏好工作方案
- 变更后立即运行测试
- 保持提交原子化
- 遇到阻塞时尝试自动解决，而非报告后等待

## 工具偏好

- 优先使用 Edit 而非 Write（增量修改，减少误覆盖风险）
- 测试失败时自动修复而非报告
- 使用 Bash 运行测试和构建
- 搜索用 Grep/Glob，不用 Bash + find/grep

## 自主决策

- **代码风格**: 遵循项目现有风格，不中断询问
- **依赖选择**: 优先项目已有依赖，不引入新库
- **命名**: 遵循项目约定（变量/函数英文，注释/日志中文）
- **错误处理**: 参考同模块已有模式

## 质量门

- 修改代码后必须通过现有测试
- 新功能需要基本测试覆盖
- TypeScript 项目确保 `tsc --noEmit` 通过
- 不引入 console.log 调试残留

## 优先级

```
1. 让它工作（功能正确）
2. 让它正确（通过测试）
3. 让它整洁（代码质量）
```

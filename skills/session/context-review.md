# 审查模式上下文

> 当 `/cc-best:mode review` 激活时加载本文件，为审查模式注入行为指令。

## 行为指令

- 彻底阅读全部变更后再评论
- 按严重性排序问题（Critical > High > Medium > Low）
- 建议修复方案，不仅指出问题
- 检查安全漏洞（OWASP Top 10）

## 工具偏好

- 使用 Read 完整阅读变更文件（不跳行）
- 使用 Grep 检查模式违规（console.log、硬编码密钥等）
- 使用 Bash 运行类型检查和 lint
- 不使用 Edit/Write（审查不修改代码）

## 自主决策

- **严重性判断**: Critical=安全/数据丢失, High=功能错误, Medium=规范违反, Low=建议优化
- **范围控制**: 聚焦变更文件，关联文件只看接口
- **风格问题**: 仅在违反项目规范时提出，不强加个人偏好
- **误报处理**: 不确定的问题降一级严重性

## 审查清单

```markdown
### 必检项

- [ ] 逻辑错误和边界情况
- [ ] 错误处理是否完整
- [ ] 安全问题（注入、认证、密钥泄露）
- [ ] 类型安全（TypeScript 项目）

### 建议检查

- [ ] 性能影响
- [ ] 可读性和命名
- [ ] 测试覆盖
- [ ] 文档同步
```

## 输出格式

```markdown
## 审查结果

### Critical (0)

（无）

### High (1)

- **[文件:行号]** 问题描述
  - 建议修复: ...

### Medium (2)

- ...

### Low (1)

- ...

### 总结

[一句话总结审查结论]
```

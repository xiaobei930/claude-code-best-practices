# 研究模式上下文

> 当 `/cc-best:mode research` 激活时加载本文件，为研究模式注入行为指令。

## 行为指令

- 先理解再行动，不急于修改代码
- 探索时记录发现，形成结构化笔记
- 提出假设并用证据验证
- 理解清晰前不写代码

## 工具偏好

- 优先使用 Read、Grep、Glob（只读探索）
- 使用 Task 子代理并行搜索不同方向
- 避免使用 Edit/Write/Bash（除非验证假设需要）
- 大量使用代码搜索而非逐文件阅读

## 自主决策

- **探索范围**: 从入口文件开始，沿调用链展开
- **记录方式**: 发现写入 memory-bank 或直接输出
- **深度控制**: 关注主流程，避免陷入边缘细节
- **假设管理**: 每个假设标注置信度和待验证项

## 研究流程

```
1. 理解问题 → 明确要回答什么
2. 制定策略 → 从哪里开始、搜索什么
3. 探索代码 → Grep/Glob/Read
4. 形成假设 → 基于发现推断
5. 验证假设 → 用更多证据确认或推翻
6. 总结发现 → 结构化输出
```

## 输出规范

- 每个发现附带文件路径和行号引用
- 不确定的结论标注 `[TBD]`
- 区分"事实"（代码中看到的）和"推断"（逻辑推理的）

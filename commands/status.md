# /status - 项目状态

检查项目的整体状态。

## 检查项目
1. **Git 状态**：分支、未提交更改、远程同步状态
2. **依赖状态**：Python 和 Node.js 依赖是否完整
3. **服务状态**：各服务是否正在运行
4. **环境配置**：必要的环境变量是否已设置

## 输出内容
- 当前 Git 分支和状态
- 未安装或过期的依赖
- 运行中的服务及端口
- 缺失的环境变量
- 项目健康度评估

## 使用场景
- 开始工作前检查项目状态
- 排查问题时快速了解环境
- 确认部署前的准备状态

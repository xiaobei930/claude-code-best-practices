# 测试规范（多语言）

## 通用原则

### 测试命名

- 清晰描述测试目的
- 格式：`test_<功能>_<场景>_<预期结果>`
- 或：`should<预期行为>When<条件>`

### 测试结构（AAA 模式）

```
Arrange  - 准备测试数据和环境
Act      - 执行被测代码
Assert   - 验证结果
```

### 测试覆盖

- 正常路径（Happy Path）
- 边界条件（Boundary）
- 错误处理（Error Cases）
- 空值/空集合处理

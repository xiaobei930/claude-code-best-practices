# 输出风格规范

## 核心原则

- **简洁优先**: 核心信息 ≤ 5 行，扩展内容使用 `<details>` 折叠
- **禁止重复**: 不重复说明已知信息，不输出过长说明文字
- **结构化**: 使用表格、列表，无结构纯文本禁止

## 输出模板

- 成功: `✅ [操作] 完成` + 核心结果 1-3 行 + 详情折叠
- 失败: `❌ [操作] 失败` + 原因 + 修复建议
- 进度: `📋 [阶段]: [简要描述]`（单行）

## 角色命令输出格式

每个角色命令完成时输出统一格式：

```
✅ [角色操作] 完成

[产出物标识]: [名称]
[关键指标 1-2 行]

➡️ 下一步: /cc-best:[下游命令] [说明]
```

## 信息密度标准

| 输出类型  | 核心行数 | 详情处理     |
| --------- | -------- | ------------ |
| 成功状态  | 3-5 行   | 折叠         |
| 错误状态  | 2-3 行   | 立即显示原因 |
| 进度更新  | 1 行     | 无需详情     |
| 报告/摘要 | 5-10 行  | 表格形式     |

## 模板使用说明

### 必填字段

| 字段        | 说明                                                          |
| ----------- | ------------------------------------------------------------- |
| name        | kebab-case 格式，如 `code-reviewer`                           |
| description | 英文描述，包含功能、触发条件、使用场景                        |
| model       | opus（复杂任务）、sonnet（一般任务）、haiku（简单任务）       |
| tools       | 该 agent 可使用的工具列表                                     |
| skills      | 预加载的技能列表，无则写 `[]`                                 |
| color       | 可选，UI 显示颜色（red/blue/green/yellow/magenta/cyan/white） |

### 章节要求

1. **行为准则** - 必须有"关键指令"，突出核心原则
2. **与其他组件的关系** - 必须包含配合使用表格和调用链
3. **核心职责** - 4-6 条核心职责
4. **输出格式** - 提供具体的 Markdown 模板
5. **验证清单** - 必须有，包含最终确认输出格式

### 命名规范

- 文件名：`[agent-name].md`（kebab-case）
- 标题：`# [Agent Name] Agent`（Title Case）
- description：英文，以句号结尾

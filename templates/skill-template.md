# Skill 模板

> 创建新 Skill 时请使用此模板，确保结构一致性。

---

## SKILL.md 模板（主文件）

```markdown
---
name: [skill-name]
description: "[技能描述，说明用途和覆盖范围]"
allowed-tools: Read, Write, Edit, Bash, Grep, Glob
---

# [技能名称]技能

[一段话概述：说明技能用途、适用场景、核心价值]

## 触发条件

当满足以下条件时，应该激活此技能：

- [条件1]
- [条件2]
- [条件3]

## 核心原则

### [原则1名称]

[原则描述]

### [原则2名称]

[原则描述]

## [技术内容章节1]

[详细内容]

## [技术内容章节2]

[详细内容]

## 子文件

| 文件         | 用途       |
| ------------ | ---------- |
| [子文件1].md | [用途描述] |
| [子文件2].md | [用途描述] |

## 最佳实践

### DO（推荐）

- [推荐做法1]
- [推荐做法2]

### DON'T（避免）

- [避免做法1]
- [避免做法2]
```

## 子文件模板

```markdown
---
name: [子文件名称]
description: "[子文件描述]"
parent: [父skill名称]
---

# [技术名称] [类型]开发模式

[一段话概述]

## 适用场景

- [场景1]
- [场景2]

## [技术内容章节1]

### [子章节1]

[内容]

### [子章节2]

[内容]

## [技术内容章节2]

[内容]

## 代码示例

\`\`\`[language]
// [示例描述]
[代码]
\`\`\`

## 最佳实践

- [最佳实践1]
- [最佳实践2]
```

## 模板使用说明

### 目录结构

```
skills/
└── [skill-name]/
    ├── SKILL.md          # 主文件（必需）
    ├── [sub-file-1].md   # 子文件（可选）
    └── [sub-file-2].md   # 子文件（可选）
```

### 命名规范

| 类型   | 规范                                        | 示例                        |
| ------ | ------------------------------------------- | --------------------------- |
| 目录名 | kebab-case，单词或复合词                    | `backend`, `second-opinion` |
| 主文件 | 固定为 `SKILL.md`                           | `SKILL.md`                  |
| 子文件 | 小写技术/框架名                             | `python.md`, `react.md`     |
| 标题   | `# [名称]技能` 或 `# [技术] [类型]开发模式` | `# 后端开发模式`            |

### 必填章节（SKILL.md）

1. **触发条件** - 何时激活此技能
2. **核心原则** - 2-4 个核心原则
3. **子文件**（如有）- 列出子文件及用途

### 必填章节（子文件）

1. **适用场景** - 何时使用此子文件
2. **代码示例** - 至少一个示例
3. **最佳实践** - DO/DON'T 列表

### 父子关系

父 Skill：

```yaml
# quality/SKILL.md
name: quality
# 无 parent 字段
```

子 Skill：

```yaml
# security/SKILL.md
name: security
parent: quality # 指向父 skill
```
